# K-LOL.GG 기능 검수 기준표

이 문서는 기능을 고치기 전에 “기능 목적 → 정상 기대값 → 확인 방법 → 수정 우선순위”를 고정하기 위한 기준표입니다.

## 1. 검수 원칙

| 기준 | 설명 |
| --- | --- |
| 목적 우선 | 화면이나 API가 존재하는 이유를 먼저 정의합니다. |
| DB 기준 | 화면 성공 메시지보다 DB 저장값과 재조회 결과를 우선합니다. |
| 한 흐름 검증 | 등록 → 목록 → 상세 → 통계/로그 반영까지 이어서 확인합니다. |
| 삭제 보수성 | import/route/API 호출 여부가 확인되지 않은 파일은 즉시 삭제하지 않습니다. |
| 운영 기준 | 운영자가 실제 카카오톡/디스코드/관리자 화면에서 쓰는 흐름을 우선합니다. |

## 2. 기능별 정상 기준

| 기능 | 목적 | 정상 기대값 | 1차 확인 방법 |
| --- | --- | --- | --- |
| 로그인/승인 | 승인 유저와 관리자를 분리 | 미로그인 차단, 승인 유저 접근, 관리자 기능 분리 | `/login`, `/me`, `/admin` |
| 플레이어 | 참가자 기본 정보 관리 | 이름, 닉네임#태그, 티어, 라인, 계정 연결 일치 | `/players`, `/players/[id]`, `/admin/users` |
| 내전 등록 | 세트/팀/참가자/KDA 저장 | MatchSeries, MatchGame, MatchParticipant 일관 생성 | `/admin/matches/new` → `/matches` |
| 시즌/랭킹 | 시즌별 성과 계산 | 시즌 기준 참여 수, 승률, KDA, MVP 일치 | `/rankings`, `/api/stats/**` |
| 팀 밸런스 | 티어/라인 기반 팀 구성 | 참가자 수, 라인, 저장/불러오기 일치 | `/players/balance` |
| 밴픽 추천 | 내전/솔랭 기록 기반 추천 | 추천 챔피언과 상대 밴 후보가 의도 기준에 맞음 | `/players/balance/recommendations` |
| 카카오톡 내전 | 카톡 양식 참가자 반영 | 양식 수정 시 추가/삭제 동기화, 중복 방지 | 내전 양식 전송 → `/admin/recruits` |
| 카카오톡 구인 | 파티 생성/현황/마감 | 번호, 상태, 인원, 보호시간, 마감 처리 일치 | `/2인파티`, `구인현황`, `번호ㅉ` |
| 운영 양식 | 지인/건의/모임/외출 접수 | 양식별 필수값 인식, 관리자 화면 반영 | 카톡 양식 전송 → 관리자 목록 |
| 디스코드 운영 | 음성방 이벤트/출석/자동마감 | JOIN/LEAVE, 현재 접속자, 자동마감 후보/완료 일치 | `/admin/discord` + PM2 로그 |
| 모바일 APP | 모바일 전용 핵심 기능 | `/app` 내부에서 PC 화면 이탈 없음 | `/app`, `/app/admin`, `/app/me` |
| 멸망전/경매 | 이벤트 추첨/경매 진행 | 카드/티어/추첨 연출이 중복 없이 자연스럽게 진행 | `/destruction-auction-live` |

## 3. 카카오톡 명령어 검수 순서

| 순서 | 입력 | 기대 결과 |
| --- | --- | --- |
| 1 | `봇버전` | 현재 봇 코드 버전 출력 |
| 2 | `도움말` | 전체 명령어 안내 출력 |
| 3 | `내전구인` | 내전 참가 신청 양식 출력 |
| 4 | 내전 양식에 참가자 1명 이상 입력 | 참가자 자동 등록 또는 서버 reply 출력 |
| 5 | `내전현황` | 현재 참가자 목록 출력 |
| 6 | `/2인파티` | 새 구인 번호와 양식 출력 |
| 7 | `구인현황` | 진행 중 구인 목록 출력 |
| 8 | 구인 양식 복사 후 이름 추가 | 해당 번호 인원 증가 |
| 9 | `번호ㅉ` | 해당 구인 마감 |
| 10 | 지인/건의/모임/외출 양식 | 관리자 운영 양식 목록에 반영 |

## 4. 디스코드 검수 순서

| 순서 | 확인 | 기대 결과 |
| --- | --- | --- |
| 1 | `pm2 list` | `klol-discord-operation-server` online |
| 2 | 음성방 입장 | `VOICE:JOIN` 또는 관리자 현재 접속자 반영 |
| 3 | 음성방 퇴장 | `VOICE:LEAVE` 반영 |
| 4 | 구인 파티와 음성방 비교 | 관리자 구인 검증에 접속/미접속/외부인 일치 |
| 5 | 기준치 미달 후 hold 경과 | 자동마감 후보 또는 완료 처리 |
| 6 | 내전 시작 전후 | 지각/불참 기준 적용 |

## 5. 삭제 판단 기준

| 분류 | 처리 |
| --- | --- |
| `.bak`, `_backup`, `temp`, `upload` | import/실행 참조가 없으면 삭제 후보 |
| `.next`, `node_modules`, `tsconfig.tsbuildinfo` | Git/검수 ZIP 제외, 로컬 삭제 가능 |
| `.env`, `.env.local`, `.env_backup*` | 공유/커밋 금지, 백업 파일은 삭제 권장 |
| 오래된 CSS | `globals.css`, component import, className 사용 여부 확인 후 삭제 |
| API route | 화면/봇/서버 호출 여부 확인 전 삭제 금지 |
| Prisma model | 마이그레이션/운영 DB 영향 때문에 단독 삭제 금지 |

## 6. 자동 리포트 생성

패치 적용 후 아래 명령어로 현재 프로젝트 구조 리포트를 생성합니다.

```powershell
npm run audit:project
```

생성 파일:

```txt
docs/audit/generated/KLOL_PROJECT_AUDIT.md
docs/audit/generated/KLOL_PROJECT_AUDIT.json
```
