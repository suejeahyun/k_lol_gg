# K-LOL.GG 팀 밸런스 AI/MMR 개선 요약

## 반영 범위

- 최고티어 60% / 현재티어 30% / 내전지표 10% 기본 점수 유지
- 솔로랭 최근 20게임 폼 보정
- 솔로랭 주/부포지션 추정
- 신청 주/부포지션 비교 보정
- 라인별 영향도 배율 적용
- 주/부/AUTO 감점
- 다이아2 이상/마스터 이상 고티어 주포지션 우선
- 총합 균형형 / 라인 균형형 / 포지션 만족형 3안 유지
- 바텀 조합, 미드-정글 조합 위험도 반영
- 결과 카드 설명 강화
- 솔로랭 갱신 결과 표시 유지
- 관리자 수동 보정 반영 유지
- 계산 품질 점수, 경고 메시지, 추천안 자동 표시 추가
- 내전 후 밸런스 피드백 저장 API 추가
- 내부 AI/MMR 시스템 추가

## 신규 DB 모델

- PlayerBalanceProfile
  - 플레이어 전체 MMR, 라인별 MMR, 신뢰도, 분석 경기 수 저장
- BalanceMatchReview
  - 선택한 팀 밸런스 안, 예상/실제 결과, 품질 지표, 운영자 피드백 저장
- PlayerBalanceMatchResult
  - 내전 등록 후 플레이어별 KDA/승패/포지션/MMR 변동 저장

## 동작 흐름

1. 팀 밸런스 계산 시 기존 티어/내전/솔로랭/포지션/수동보정에 내부 MMR 보정을 추가한다.
2. 계산 결과는 품질 점수와 경고 메시지를 함께 반환한다.
3. 내전 등록이 완료되면 각 플레이어의 전체 MMR과 라인별 MMR을 자동 업데이트한다.
4. 관리자는 `/api/team-balance/feedback`로 내전 후 밸런스 피드백을 저장할 수 있다.

## 배포 전 명령

```powershell
npx prisma generate
npx prisma migrate deploy
npm run lint
npm run build
```

## 주의

- 신규 Prisma 모델이 있으므로 반드시 `npx prisma migrate deploy`를 먼저 실행한다.
- MMR은 초기값 50에서 시작하며, 내전 등록 데이터가 쌓일수록 신뢰도와 영향력이 증가한다.
- Riot API 실패는 팀 밸런스 계산 실패로 이어지지 않고 기존 저장 데이터를 사용한다.
